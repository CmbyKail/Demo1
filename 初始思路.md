# 情商训练软件 (EQ Trainer) - 初始思路与深度规划

## 1. 核心需求分析

用户希望构建一个情景化情商训练系统，核心价值在于：
- **应急能力训练**：高压场景下的快速反应（限时2分钟）。
- **表达能力提升**：通过练习获得AI反馈。
- **知识库结合**：融入曾仕强先生的智慧。
- **个性化训练**：多场景、多维度的系统性提升。

**关键约束条件**：
- 编程小白，需要技术栈简单易上手。
- 本地使用，数据隐私可控。
- 成本可控，避免过度设计。
- 集成AI API。

## 2. 场景体系设计

建议构建6大场景类别：

1.  **💼 职场场景**：老板批评、同事甩锅、薪资谈判、办公政治。
2.  **❤️ 情感场景**：约会、关系维护、家庭矛盾、友谊危机。
3.  **👨‍👩‍👧‍👦 家庭场景**：亲子关系、婆媳矛盾、长辈沟通、家族聚会。
4.  **🎓 学术场景**：导师关系、同学竞争、演讲答辩、学术不端指控。
5.  **🤝 社交场景**：破冰、冲突调解、拒绝艺术、危机公关。
6.  **🚨 突发场景**：公共场合纠纷、网络暴力、健康危机。

## 3. 曾仕强知识库集成策略

**方案A：关键词匹配（适合小白）**
- 预处理：TXT按主题打标签。
- 检索：关键词匹配。
- 成本：0元。

**方案B：向量数据库（专业方案 - 未来目标）**
- 工具：ChromaDB / Qdrant / FAISS。
- 流程：语义相似度搜索。
- 优势：精准匹配。

**方案C：AI总结（平衡方案 - MVP推荐）** ⭐
- **预处理**：让GPT先总结曾仕强语录的"核心原则"。
- **应用**：每次提问注入相关原则（而非全部）。
- **优势**：减少Token消耗，保持智慧精髓。
- **成本**：仅一次性的预处理成本。

## 4. 技术实现方案对比

**推荐方案：方案一（纯前端零依赖）** ⭐⭐⭐⭐⭐

- **技术栈**：HTML5 + CSS3 + 原生JavaScript。
- **AI集成**：OpenAI API / 文心一言API / 通义千问API。
- **存储**：LocalStorage（本地存储进度）。
- **优势**：零学习成本，直接浏览器运行，无服务器成本，数据隐私安全。

**架构设计**：
- **用户界面层**：场景选择、倒计时、答题区、进度统计。
- **业务逻辑层**：场景生成、计时器、评分引擎。
- **AI服务层**：API封装、提示词工程（注入曾仕强智慧）。
- **数据存储层**：LocalStorage保存历史记录和设置。

## 5. 实施路线图

1.  **Week 1 (MVP)**：基础框架（HTML/JS），场景展示，答题界面。
2.  **Week 2 (AI集成)**：集成API，实现评分反馈，计时器功能。
3.  **Week 3 (知识库)**：整理语录，优化提示词工程（方案C）。
4.  **Week 4 (完善)**：进度统计，成就系统，扩充场景库。
